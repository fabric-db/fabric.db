# Commercial Model.

FabricDB follows an open-core commercial model.

The core model, vocabulary, and public protocol contracts may be open.

Commercial products, managed services, enterprise profiles, hosted control planes, implementation packages, support, certification, and proprietary extensions may be commercial.

## Core boundary.

Open core includes.

- Fabric vocabulary.
- Core definitions.
- Box model.
- Node model.
- Agreement model.
- Tuplet model.
- Fabric Loop model.
- Mutual agreement contract.
- Standards mapping.
- Public conformance principles.

## Commercial boundary.

Commercial offerings may include.

- Managed FabricDB.
- Enterprise FabricDB runtime.
- Certified profiles.
- Hosted policy/control plane.
- Enterprise governance packs.
- Commercial connectors.
- Commercial UI builder.
- Commercial observability and audit console.
- Support and SLAs.
- Certification services.
- Deployment automation.
- Security hardening packs.

## Rule.

```text
Open core defines the model.
Commercial offerings operationalize the model.
```

## License posture.

Do not assume all implementation code is Apache-2.0.

The repository may contain open documentation and core protocol definitions while reserving commercial rights for products and services built around the model.

## No confusion rule.

The core must clearly mark what is open and what is commercial.

Commercial artifacts must not be presented as open-source core unless explicitly licensed that way.

## Final statement.

```text
FabricDB is open-core.
The market implementation is commercial.
```
